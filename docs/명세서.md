# 프로젝트 명세서

## 단어 벡터를 통한 문자-색 공감각의 시각화

---

## Topic & Motivation

문자-색 공감각(Grapheme-Color Synesthesia)은 글자나 단어를 볼 때 특정 색상이 자동으로 연상되는 인지 현상입니다. 인구의 약 1~2%가 경험하는 이 현상을 언어 모델에 적용해서, 모델이 학습한 단어 공간에 이미 색채 구조가 내재되어 있는지 탐구합니다.

**Novelty 한 문장:** 텍스트만으로 학습한 언어 모델의 임베딩 공간에서 인간 공감각과 유사한 색채 구조를 추출하고, 이를 텍스트↔이미지 양방향 변환의 예술적 도구로 활용한다는 점에서 기존 공감각 연구와 NLP 해석 가능성 연구를 동시에 확장합니다.

---

## Task Definition

### 정방향 (텍스트 → 시각화)

**입력**

- 텍스트 (시, 문장, 자유 텍스트)
- 사용자 파라미터: 공감각 비중 β (0~1), 노이즈 강도 γ, 해상도

**출력**

- 토큰별 색상 바
- 2D 색상 이미지 (한 단어 = 한 픽셀, 서브워드는 평균 풀링)
- 3D 색상 타워 (XY: RGB 삼각형 좌표, Z: 시간/순서)

**구체적 예시**

```
입력: "The night was silent and cold"
      β=0.5

출력 (토큰별 색상):
The      → ██ (베이지)
night    → ██ (짙은 남색)
was      → ██ (회색)
silent   → ██ (흰/연보라)
and      → ██ (회색)
cold     → ██ (차가운 파랑)

2D 이미지: 6픽셀이 위 색상으로 배치된 이미지
3D 타워: 각 단어가 RGB 삼각형 위 한 점으로 Z축으로 쌓임
```

### 역방향 (이미지 → 텍스트)

**입력**

- 이미지 (JPG/PNG)
- 선택적 키워드 (기본값: 없음, α=0)

**출력**

- 픽셀 격자 구조를 유지한 구조적 시 (날것 출력)

**구체적 예시**

```
입력: 노을 사진 (붉은/주황 계열)
      키워드: 없음 (α=0)
      다운샘플링: 8×8 → 64픽셀

출력:
burn   flame  glow   heat
ember  dusk   fade   amber
...
(8줄 × 8단어, 이미지 격자 구조 유지)
```

### 순환 실험

```
시 입력 → [정방향] → 색상 이미지 → [역방향] → 새로운 시
```

원본 시와 순환 후 생성된 시를 비교해서 모델이 의미를 색으로, 색을 다시 의미로 얼마나 잘 번역하는지 보여줍니다.

## Approach

### 전체 아키텍처

> **[교수님 피드백 반영]**  
> **피드백 1:** BERT 사용 방식 명확화 — 단어를 단독 입력(Isolated)하여 고정 벡터 추출  
> **피드백 2:** Linear Probe 검증 실험 결과 반영 — 과적합 확인, 색상 정확도 55% → MLP로 대체  
> **피드백 3:** Linear Probe → MLP (768→32→3) 업그레이드

```
[정방향]
텍스트 입력
  ↓ BERT Tokenizer
  서브워드 처리: 여러 서브워드로 쪼개진 경우 벡터 평균 → 단어당 픽셀 1개
  예) "moonlight" → ["moon", "##light"]
      → mean(v_moon, v_##light) → 픽셀 1개
토큰 시퀀스 [token_1, token_2, ... token_n]
  ↓ BERT (bert-base-uncased) — 단독 입력(Isolated) 방식
  ※ 각 단어를 "[CLS] word [SEP]" 형태로 단독 입력
  ※ 문장 문맥 없이 단독 처리 → 같은 단어는 항상 동일 벡터 (정적 임베딩과 동일)
768차원 벡터 (토큰별)
  ↓ Mean Centering 적용
  v_i = BERT(token) - mean_vec
  (mean_vec: seed 30개 + 시 코퍼스 상위 190개 = 220개로 계산)
  ↓
┌──────────────────────────────────────────────────┐
│                 색상 계산 레이어                  │
│                                                  │
│  방법 1. MLP  [교수님 피드백 반영 - 피드백 3]    │
│  구조: 768 → 32 → 3                              │
│  v_i (768차원)                                   │
│    → Linear(768→32) + ReLU + Dropout(0.3)        │
│    → Linear(32→3) + Sigmoid × 255               │
│    → RGB_syn (0~255)                             │
│  학습 데이터 구성:                               │
│  입력: BERT(word) 단독 - mean_vec  (768차원)     │
│  정답: RGB (0~255)                               │
│  - Eagleman: BERT("A")~BERT("Z") → 26쌍         │
│  - NRC: BERT("fire") 등 → 11,449쌍              │
│  합계: 11,475쌍                                  │
│                                                  │
│  방법 2. Cosine Similarity                       │
│  (BERT + Mean Centering, 768차원)                │
│  ※ 앵커 벡터는 1회 계산 후 .npy로 저장          │
│  A_R = mean([BERT(w)-mean_vec for w in seeds_r]) │
│  A_G = mean([BERT(w)-mean_vec for w in seeds_g]) │
│  A_B = mean([BERT(w)-mean_vec for w in seeds_b]) │
│                                                  │
│  seeds_r: red, crimson, scarlet, blood, flame,  │
│           fire, rose, ruby, sunset, passion      │
│  seeds_g: green, emerald, forest, grass, leaf,  │
│           moss, fern, meadow, nature, jungle     │
│  seeds_b: blue, sapphire, ocean, sky, sea,      │
│           water, river, twilight, indigo, cobalt │
│                                                  │
│  RGB_uni = normalize(                            │
│    cos(v_i, A_R), cos(v_i, A_G), cos(v_i, A_B)) │
│                                                  │
│  최종 혼합                                       │
│  RGB_final = β*RGB_syn + (1-β)*RGB_uni          │
│                                                  │
│  알파벳 노이즈                                   │
│  RGB_out = RGB_final                             │
│          + γ*(RGB_grapheme - RGB_final)          │
│                                                  │
│  RGB_grapheme 정의:                              │
│  토큰의 첫 글자 Eagleman 색상 사용               │
│  예) "night" → 첫 글자 N                        │
│      → Eagleman["N"] = (140, 102, 63)            │
│  근거: 공감각 연구에서 단어 색상은               │
│  첫 글자 색상이 지배적                           │
│  (Rich et al., 2005; Simner et al., 2006)        │
└──────────────────────────────────────────────────┘
  ↓                        ↓
2D 이미지               3D 타워
(raster scan)         (RGB삼각형 + Z축)

[역방향]
이미지 입력
  ↓ 다운샘플링 (ex. 16×16)
픽셀 단위 (R, G, B)
  ↓ Anchor 기반 역변환 (0~1 정규화 후 사용)
  v_i = (R/255)*A_R + (G/255)*A_G + (B/255)*A_B
  ↓ (선택) 키워드 문맥 혼합 (기본값 α=0)
  v_word = BERT(word) - mean_vec  ← 후보 단어도 동일하게 Mean Centering 적용
  score(word) = α*cos(v_word, v_context)
              + (1-α)*cos(v_word, v_i)
  ↓ softmax 샘플링 (후보: 시 코퍼스 9,942개 단어)
구조적 시 (날것 출력, LLM 후처리 없음)
```

### Cosine Similarity — BERT Mean Centering 검증 결과

BERT는 anisotropy 문제로 인해 raw 벡터만으로는 색상 구분이 불가능합니다. Mean Centering 적용 후 94% 색상 정확도를 확인했습니다.

| 방법 | 앵커 간 코사인 유사도 | 색상 정확도 | 채택 |
| --- | --- | --- | --- |
| Raw BERT | 0.89~0.93 (구분 불가) | 시각적 회색 수렴 | ✗ |
| BERT Whitening | 0.06~0.66 (불안정) | 62% | ✗ |
| **BERT + Mean Centering** | **0.23~0.47 (양호)** | **94%** | **✓** |

앵커 벡터 색상 직관성 검증 (17/18 통과):

| 단어 | RGB | 기대 색상 | 결과 |
| --- | --- | --- | --- |
| fire | (105, 70, 78) | 빨강 | ✓ |
| ocean | (73, 76, 104) | 파랑 | ✓ |
| forest | (71, 98, 85) | 초록 | ✓ |
| night | (90, 66, 97) | 어둠/파랑 | ✓ |
| snow | (85, 85, 84) | 흰/밝음 | ✓ |
| shadow | (92, 71, 91) | 어둠 | ✗ (경계선) |

### 모델 선택 이유

**BERT를 단독 입력(Isolated) 방식으로 사용하는 이유** `[교수님 피드백 반영 - 피드백 1]`는 단일 알파벳 문자도 의미 있는 벡터를 뽑을 수 있고, 정방향과 역방향이 동일한 768차원 임베딩 공간을 공유해서 양방향 변환의 일관성이 유지되기 때문입니다. 단어를 `[CLS] word [SEP]` 형태로 단독 입력하므로 같은 단어는 항상 동일한 벡터를 반환합니다.

**MLP(768→32→3)를 사용하는 이유** `[교수님 피드백 반영 - 피드백 3]`는 "임베딩 공간에 색채 정보가 존재하는가"라는 프로젝트의 핵심 질문에 직접적인 답을 주기 때문입니다. 검증 실험 결과 단순 Linear Probe(768→3)는 학습 셋 L2=10.9 / 테스트 셋 L2=193.4의 심각한 과적합을 보였으므로 중간 레이어(32차원)와 Dropout(0.3)을 추가한 MLP로 업그레이드합니다. Eagleman + NRC 총 11,475쌍으로 학습합니다.

**BERT Mean Centering을 적용하는 이유**는 BERT 벡터의 anisotropy 문제로 인해 raw 벡터만으로는 색상 방향이 구분되지 않기 때문입니다. seed 단어 30개 + 시 코퍼스 상위 190개로 계산한 mean_vec을 빼줌으로써 앵커 벡터 간 코사인 유사도가 0.89~0.93에서 0.23~0.47로 개선됩니다.

**Cosine Similarity를 병행하는 이유**는 정답지 없이 모델 스스로 형성한 색채 구조를 보여주기 위해서입니다. β 슬라이더로 두 방식을 실시간 비교할 수 있어 인간 공감각 vs. AI 세계관이라는 핵심 대비를 직관적으로 보여줍니다.

**Anchor 기반 역변환을 사용하는 이유**는 역 Linear Probe(768→3 역변환)는 정보 손실이 크고 불안정하기 때문입니다. Anchor 기반은 정방향과 동일한 벡터 공간을 사용해서 정방향-역방향 사이의 대칭성이 유지됩니다.

- MLP 학습 (768→32→3, NRC 11,449쌍 + Eagleman 26쌍)
- Cosine Similarity 색상 계산 (A_R / A_G / A_B 앵커 기반)
- β 혼합 (`RGB_final = β*RGB_syn + (1-β)*RGB_uni`) 및 γ 알파벳 노이즈 적용
- 서브워드 평균 풀링 처리 (예: "moonlight" → mean(v_moon, v_##light))
- 결과물: `forward_pipeline.py`, 학습된 MLP 가중치 (`mlp_weights.pt`)

### B — 역방향 파이프라인 (이예원)

- 이미지 다운샘플링 (PIL, 8×8 / 16×16 / 32×32 선택)
- Anchor 기반 역변환 (`v_i = (R/255)*A_R + (G/255)*A_G + (B/255)*A_B`)
- 후보 단어 집합(9,942개)과의 cosine similarity 비교 → 최근접 단어 선택
- α 키워드 문맥 혼합 (`score = α*cos(v_word, v_context) + (1-α)*cos(v_word, v_i)`)
- 구조적 시 출력 (이미지 격자 구조 유지, LLM 후처리 없음)
- 결과물: `reverse_pipeline.py`

### C — 3D 시각화 & 데모 UI (김민서)

- Gradio 웹 데모 전체 구성 (탭 1: 정방향, 탭 2: 역방향)
- 2D 픽셀 이미지 생성 (PIL raster scan, 한 단어 = 한 픽셀)
- 3D 색상 타워 시각화 (Plotly, XY: RGB 삼각형 좌표, Z: 순서, hover 기능)
- β / γ / α 슬라이더 연동 및 실시간 업데이트
- 정방향 ↔ 역방향 탭 통합 및 순환 실험 시연 구성
- 결과물: `app.py` (Gradio), `visualizer.py`
