## 피드백 1: BERT 사용 방식 명확화

### 교수님 피드백
> BERT를 어떻게 사용할 것인지 구체적으로 서술할 것.  
> Contextual embedding인지, 단순 정적(static) embedding으로 사용하는 것인지 불명확하다.

### 문제점
명세서 초안에서 "BERT 768차원 벡터"라고만 기술했으며, 문맥(context)을 반영하는지 여부가 불명확했음.

- BERT는 문장 전체를 입력받아 문맥에 따라 다른 벡터를 출력하는 contextual 모델이지만
- 우리 프로젝트에서는 단어를 **단독으로** 입력해서 고정 벡터를 얻는 방식을 사용함
- 즉, "fire"는 항상 동일한 벡터를 반환 → **정적(static) 임베딩과 동일한 방식**

### 해결 방안
BERT를 **Isolated (단독 입력) 방식**으로 사용함을 명시:

```python
# 단어 단독 입력 → 고정 벡터 추출
inputs = tokenizer(word, return_tensors='pt')
with torch.no_grad():
    out = model(**inputs)
v_i = out.last_hidden_state[0, 1, :].numpy()  # [CLS] word [SEP] 중 word 위치
```

- 입력 형식: `[CLS] word [SEP]` (문장 아닌 단어 단독)
- 같은 단어는 항상 같은 벡터 → 색상 일관성 보장
- BERT를 문맥 없이 단독 사용하는 이유: NRC 학습 데이터(단어 단위)와 일치시키기 위함

### 명세서 반영 위치
`Approach > 전체 아키텍처 > BERT 사용 방식` 항목에 추가 명시

## 피드백 2: Linear Probe 색상 캡처 검증 실험

### 교수님 피드백
> Linear Probe가 실제로 색채 정보를 캡처하는지 검증 실험이 필요하다.  
> BERT 임베딩 공간에 색채 정보가 선형적으로 존재한다는 전제가 성립하는지 먼저 확인해야 한다.

### 검증 실험 설계

| 항목 | 내용 |
|---|---|
| 검증 일자 | 2026-05-06 |
| 모델 | Linear Probe (768→3, MSE Loss) |
| 학습 데이터 | NRC Word-Colour 500쌍 (랜덤 샘플) |
| 입력 벡터 | BERT (bert-base-uncased) + Mean Centering |
| 분할 | 학습 400쌍 / 테스트 100쌍 |
| 학습 설정 | 500 epochs, lr=0.001, Adam |

### 검증 실험 결과

#### 실험 1: 학습/테스트 일반화 (과적합 여부)

| 지표 | 학습 셋 | 테스트 셋 | 차이 |
|---|---|---|---|
| 평균 L2 거리 | 10.90 | **193.43** | **182.53** |

> **해석:** 학습 셋에서는 오차 10.90으로 낮은 편이나, 테스트 셋에서는 193.43으로 급등.  
> → **심각한 과적합(Overfitting)** 발생. 모델이 색채 구조를 일반화하지 못하고 학습 데이터를 암기한 것으로 보임.

#### 실험 2: 색상 방향 정확도

| 지표 | 값 |
|---|---|
| 테스트 셋 색상 방향 정확도 (R/G/B 채널 중 지배 채널 일치) | **55/100 (55%)** |

> **해석:** 55%는 세 색상(R/G/B) 중 무작위 선택 시 기대값(33%)보다는 높지만,  
> 색채 구조를 진정으로 학습한 모델에게 기대되는 수준(80%+)에는 크게 못 미침.  
> → Linear Probe(768→3 단순 선형)만으로는 색채 정보를 충분히 캡처하지 못함.

#### 실험 3: Eagleman Ground Truth 외부 검증

| 항목 | 값 |
|---|---|
| 알파벳 A~Z 예측 vs. Eagleman 정답 평균 L2 | **182.53** |
| 최대 L2 (I, Y) | 349.12 |
| 최소 L2 (D) | 28.7 |

> **해석:** NRC 단어 데이터로 학습한 모델이 알파벳 수준 Eagleman 데이터에 전혀 일반화되지 않음.  
> 학습 도메인(단어)과 평가 도메인(알파벳) 간 격차가 크며, 학습 쌍이 부족하고 선형 구조가 한계임.

#### 실험 4: 추상 단어 색상 예측

| 단어 | 예측 RGB | 지배 채널 | RGB 최대 편차 |
|---|---|---|---|
| democracy | (277, 182, 117) | R | 159.8 |
| algorithm | (208, 65, 44) | R | 164.1 |
| ambiguity | (81, 56, 155) | B | — |

> **해석:** 색상이 명확하지 않은 추상 단어에서 예측값이 0~255 범위를 벗어나거나(277),  
> RGB 채널 간 편차가 150+ 수준으로 과도하게 분리됨.  
> → 모델이 학습 데이터의 spurious correlation(우연한 패턴)을 학습했을 가능성 높음.

### 결론 및 해결 방안

| 결론 | 해결 방안 |
|---|---|
| 768→3 단순 선형 레이어는 과적합 & 일반화 실패 | **MLP로 업그레이드 (피드백 3)** |
| 11,475쌍도 과적합 방지에 불충분할 수 있음 | Dropout 추가 + 데이터 증강 검토 |
| BERT 임베딩에 색채 정보는 존재하나 비선형적 구조일 가능성 | 768→32→3 MLP로 비선형성 추가 |

> **피드백 2 검증 실험의 시사점:**  
> 교수님의 우려가 실험으로 확인됨. Linear Probe 단독으로는 한계가 있으며,  
> 이로 인해 Cosine Similarity 방식(방법 2)과의 β 혼합이 더욱 중요한 설계 선택이 됨.  
> 방법 1(MLP) 혼자서는 색채를 완벽히 설명할 수 없고, 방법 2(Cosine Similarity)가 보완 역할을 한다는 것이 프로젝트의 구조적 논점이 됨.

## 피드백 3: Linear Probe → MLP 업그레이드

### 교수님 피드백
> 768→3 단순 선형 레이어(Linear Probe) 대신 중간 레이어를 추가한 MLP를 사용할 것.  
> 선형 변환만으로는 임베딩 공간의 색채 구조를 충분히 추출하지 못할 수 있다.

### 문제점
명세서 초안의 Linear Probe 구조:
```
v_i (768차원) → W (768×3) + b (3) → RGB (3차원)
```
- 선형 변환 1회만 수행 → 표현력 부족
- 피드백 2 검증 실험에서 과적합 확인 → 구조 복잡도 증가가 필요

### 해결 방안: MLP (768→32→3)

```
v_i (768차원)
  ↓ Linear(768→32) + ReLU + Dropout(0.3)
  hidden (32차원)
  ↓ Linear(32→3) + Sigmoid × 255
  RGB (3차원, 0~255)
```

| 항목 | Linear Probe (이전) | MLP (수정) |
|---|---|---|
| 레이어 수 | 1 | 2 |
| 구조 | 768→3 | 768→32→3 |
| 활성화 함수 | 없음 | ReLU (hidden), Sigmoid×255 (출력) |
| 정규화 | 없음 | Dropout(p=0.3) |
| 표현력 | 선형 변환만 | 비선형 색채 패턴 학습 가능 |

### 기대 효과
- 비선형 패턴(예: 특정 의미 클러스터가 빨강에 가깝다는 복잡한 관계) 학습 가능
- Dropout으로 과적합 완화
- 여전히 소규모 모델 → 해석 가능성 유지 (딥러닝 블랙박스와 차별화)

### 명세서 반영 위치
`Approach > 전체 아키텍처 > 방법 1` 항목: Linear Probe → MLP로 수정
