# 구현 디벨롭 노트 — 명세 대비 발전 사항

> **목적:** 명세서(`docs/명세서.md`) 작성 이후, 실제 구현 과정에서 팀원들이
> 의도적으로 발전시키거나 변경한 사항을 정리한다. 최종 보고서 및 발표 자료에
> "왜 이렇게 구현했는가"를 설명하는 근거로 사용한다.
>
> 모든 항목은 명세와의 단순 불일치가 아니라 **의도된 개선**이다.

---

## 1. `rgb_uni` 색상 계산 — 코퍼스 통계 기반 채널 편향 보정 (정방향)

**명세:**
```
RGB_uni = normalize(cos(v_i, A_R), cos(v_i, A_G), cos(v_i, A_B)) × 255
```
단순히 세 앵커와의 코사인 유사도를 정규화하고 255를 곱하는 방식.

**구현 (`forward_pipeline.py: rgb_uni` / `load_cosine_calibration` /
`calibrated_anchor_strengths`):**
```
1. raw_cos = cos(v_i, A_R/G/B)
2. (코퍼스 1회 보정) candidate_vectors.npy 전체에서 채널별 코사인의
   평균(channel_mean)·표준편차(channel_std)를 구하고,
   각 채널이 1/3씩 지배하도록 dominance_bias를 반복 추정
3. score = (raw_cos − channel_mean) / channel_std − dominance_bias
4. color_shares = softmax(score / T),  T = 2.0     # 채널 비중
5. brightness = sqrt(mean(clip((raw_cos + 1)/2)))   # 전반 명도(원본 코사인 기준)
6. RGB = (color_shares / max) × brightness × 255 × 0.92,  clip(0, 255)
```

**이유·효과:**
- BERT anisotropy를 Mean Centering으로 완화해도, 채널별 코사인의 기준선이
  서로 달라(예: 특정 채널이 코퍼스 전반에서 항상 높음) 단순 정규화 시
  **대부분의 단어가 같은 색으로 쏠리는** 편향이 있었다.
- 코퍼스 전체의 **평균·표준편차로 각 채널을 표준화**하고, 어느 색도 구조적으로
  우세하지 않도록 `dominance_bias`를 학습해 편향을 제거한다. 그 위에서
  softmax로 비중을 정하므로 단어 고유의 색 방향만 남는다.
- 명도는 원본 코사인 평균으로 따로 계산해 "그 단어가 색 앵커들과 얼마나
  강하게 정렬되는가"라는 신호를 보존한다.
- `candidate_vectors.npy`가 없는 환경에서는 코퍼스에서 측정해 둔 fallback
  상수(`_COSINE_FALLBACK_*`)로 동일 동작을 재현한다.

> 참고: 이전 구현은 비율 제곱 + `×6` 노출 증폭으로 채도를 끌어올리는 방식이었으나,
> 채널 간 구조적 편향을 근본적으로 잡지 못해 위의 통계 기반 보정으로 교체했다.

## 2. 글자별 그레인(grain) — 단어 내부 글자 단위 색 표현 (정방향)

**명세:** γ 알파벳 노이즈는 단어 **첫 글자**의 Eagleman 색만 반영
(`RGB_out = RGB_final + γ(RGB_grapheme − RGB_final)`), 단어 = 픽셀 1개.

**구현:** γ(첫 글자 끌림)에 더해 `grain_amount` 파라미터를 신설하여
**단어 안의 각 글자**를 자신의 Eagleman 색으로 블렌딩한다
(`apply_grain_to_word` → `char_rgbs`).

**이유·효과:**
- 명세의 "한 단어 = 한 픽셀"은 정보 밀도가 낮아 2D 이미지가 단조로웠다.
- grain을 통해 **글자 단위 1픽셀** 모드를 지원, 같은 단어도 구성 문자에 따라
  내부 질감(그레인)이 생겨 문자-색 공감각이라는 주제를 시각적으로 더 직접
  드러낸다.
- `grain_amount=0`이면 기존 단어 단위와 동일 → 하위 호환 유지.

## 3. 2D 이미지 — 정사각형 그리드 + 글자/단어 단위 토글 (시각화)

**명세:** raster scan, `ncols` 지정, 부족분 회색(128,128,128) 패딩, 한 단어=한 픽셀.

**구현 (`visualizer.py: make_2d_image`):**
- `unit` 인자로 **글자별 / 단어별** 픽셀 단위 선택 (UI 라디오 버튼).
- `ncols` 슬라이더 제거 → 데이터 개수 L의 `ceil(√L)`로 **N×N 정사각형** 자동 구성.
- 패딩 색을 회색 → **검정(0,0,0)** 으로 변경.
- 1024×1024 고정, `NEAREST` 리샘플로 픽셀 아트 경계 유지.

**이유·효과:**
- 정사각형 자동 배치로 텍스트 길이와 무관하게 항상 균형 잡힌 이미지 출력.
- 검정 패딩은 어두운 전시 테마(다크 UI)와 이어져 빈 칸이 자연스럽게 묻힌다.
- 단위 토글로 발표 중 "글자 질감"과 "단어 흐름"을 즉석에서 전환 시연 가능.

## 4. 후보 단어 선택 — softmax 확률 샘플링 고정 (역방향)

**명세:** `argmax(score)` 또는 softmax 샘플링.

**구현 (`reverse_pipeline.py: find_nearest_word`):**
softmax 샘플링으로 확정, `temperature=0.5`, `np.random.choice`로 단어 선택.

**이유·효과:**
- argmax는 같은 색 → 항상 같은 단어 → 격자 시가 단조롭게 반복된다.
- 확률 샘플링은 비슷한 색에서도 다양한 시어를 뽑아 **매 실행마다 다른 시**가
  생성된다. "색을 의미로 번역"하는 예술적 변주를 강조하는 선택.
- temperature 0.5는 색 충실도와 다양성의 절충값(낮을수록 색에 충실).

## 5. 역방향 해상도 옵션에 10×10 추가 (UI)

**명세:** 8×8 / 16×16 / 32×32.

**구현:** 8×8 / **10×10** / 16×16 / 32×32, 기본값 **10×10**.

**이유·효과:**
- 8×8(64단어)은 시가 너무 짧고, 16×16(256단어)은 화면을 넘쳐 가독성이 떨어진다.
- 10×10(100단어)이 한 화면에 들어오면서도 충분한 시적 밀도를 줘 기본 시연값으로 적합.

---

## 6. 역방향 부가 출력 — 단순화 이미지 + 색상·단어 매핑표 (UI)

**명세:** 구조적 시(텍스트)만 출력.

**구현 (`reverse_pipeline.py: run_reverse_with_details`, `app.py`):**
시 텍스트에 더해 ① 다운샘플된 픽셀을 확대한 **단순화 색상 이미지**,
② (위치·색상 스와치·RGB·HEX·생성 단어) **매핑 테이블**을 함께 표시.

**이유·효과:**
- "이 색이 → 이 단어가 됐다"는 변환 과정을 관람객이 눈으로 추적할 수 있어,
  역방향 파이프라인이 블랙박스가 아님을 보여준다(해석 가능성 강조).

## 7. 슬라이더 실시간 리블렌딩 — BERT 재추론 없이 즉시 갱신 (성능)

**명세:** β/γ/α 슬라이더 연동 및 "실시간 업데이트".

**구현 (`forward_pipeline.py: reblend_forward_results`, `app.py: gr.State`):**
최초 1회만 BERT 추론 후 `rgb_syn`/`rgb_uni`를 `gr.State`에 캐싱.
β/γ/grain 슬라이더 조정 시 저장된 값에서 `blend`만 재계산해 즉시 반영.

**이유·효과:**
- 슬라이더를 움직일 때마다 BERT를 재호출하면 수 초씩 지연 → 실시간성이 깨진다.
- 모델 출력(syn/uni)은 β/γ와 무관하므로 캐싱해두면 혼합만 다시 하면 된다.
  발표 중 β 슬라이더로 "인간 공감각 ↔ AI 세계관"을 끊김 없이 시연하는 핵심 최적화.

## 8. 안정성·재현성 보강 (운영)

명세에 없던 방어 코드로, 시연 중 갑작스러운 실패를 막는다.

| 항목 | 위치 | 내용 |
|---|---|---|
| anchor 무결성 자가검증 | `load_anchors` / `_assert_anchors_centered` | mean-centering 안 된 앵커(앵커 간 cos>0.8) 로드 시 즉시 에러 |
| 필수 데이터 친절 안내 | `_require_data_files` / `preflight_check` | 누락 파일을 역할과 함께 한 번에 안내 |
| MLP 가중치 graceful fallback | `run_forward` | `mlp_weights.pt` 없으면 `rgb_syn`을 `rgb_uni`로 대체하고 경고만 |
| BERT 벡터 캐싱 | `load_*` `@cache`, `candidate_vectors.npy` | 반복 추론·재로딩 비용 제거 |
| 학습 실패 단어 제외 | `train_mlp.py: compute_bert_vectors` | 벡터화 실패 단어를 valid 마스크로 학습에서 배제 |

## 9. 역방향 후보 단어 — 고유명사·인명 제외 (데이터)

**명세:** Gutenberg Poetry 코퍼스에서 추출한 후보 단어 9,942개.

**구현 (`data/poetry_candidate_words.txt`, `data/candidate_vectors.npy`):**
고유명사 및 인명을 제거하여 **9,770개**로 축소(벡터 캐시도 9,770×768로 재생성).

**이유·효과:**
- 역방향은 색→단어 코사인 최근접으로 시어를 고르는데, 후보에 인명·지명 같은
  고유명사가 섞여 있으면 색과 무관하게 대문자 고유명사가 뽑혀 시의 결속이 깨졌다.
- 일반 시어만 남겨 색-의미 대응이 더 일관되게 작동하고, 격자 시의 분위기가 살아난다.

## 10. Semantic Space — 단어 의미 공간 3D 그래프 신설 (시각화)

**명세:** 정방향 출력은 색상 바 + 2D 이미지 + 3D 컬러 타워.

**구현 (`visualizer.py: make_semantic_space`, `forward_pipeline.py: _semantic_coordinates`):**
BERT 벡터를 PCA(SVD)로 3차원 투영해 단어를 의미 공간에 배치하고, 각 점을
**그 단어의 최종 출력색(`rgb_out`)으로 채색**한다. 최근접 이웃끼리 옅은 엣지로
이어 의미적 군집을 드러낸다.

**이유·효과:**
- "의미가 가까운 단어가 색도 비슷한가?"를 한 화면에서 직관적으로 보여준다 —
  BERT 임베딩에 색채 구조가 내재되어 있다는 프로젝트 핵심 주장을 시각적으로 뒷받침.
- 컬러 타워(시간축 색 나열)와 달리 **의미 구조 자체**를 색으로 입혀, 색과 의미의
  상관을 관람객이 눈으로 확인할 수 있다.

## 부록: UI 라벨 표기 차이

| 슬라이더 | 명세 표현 | 구현 라벨 |
|---|---|---|
| β | 인간 공감각(β=1) ↔ AI 세계관(β=0) | 색 모델 (Cosine ←→ MLP) |
| γ | 노이즈 강도 | 첫 글자 색 끌림 |
| grain | (신설) | 글자 그레인 강도 |

β의 의미는 동일하다(β=1 → MLP=`rgb_syn`, β=0 → Cosine=`rgb_uni`). 구현 라벨은
관람객이 두 색 계산 방식을 기술적으로 구분하기 쉽도록 표기를 바꾼 것이다.
